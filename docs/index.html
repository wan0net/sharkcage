yeet

Yeet your code tasks at whatever hardware you have lying around and walk away.

GitHub Documentation

The pitch

You have tasks. You have small computers. You yeet the tasks at the computers.

The computers run AI coding agents -- Claude Code, Crush, Aider -- autonomously. You go get coffee. Or sleep. Or touch grass. Whatever people do when they're not babysitting AI.

When you come back, there are draft PRs waiting for you. Reviewed, tested, ready to merge. Like a very cheap, very quiet development team that never asks about sprint velocity.

If something scary happens -- like the AI wanting to rm -rf / -- yeet pauses and texts you. Because even reckless abandonment has limits.

The stack

What you'd expect

  • Kubernetes
  • 47 microservices
  • A dedicated DevOps team
  • $2,000/month in cloud bills
  • A Slack channel called #incidents

What yeet uses

  • Nomad (single binary, one server)
  • A shell script and a job template
  • Whatever PCs you have lying around
  • ~$5/year in electricity per node
  • About 200 lines of CLI code

Orchestration: Nomad. Networking: Tailscale. Provisioning: Ansible. Device routing: udev + flock. Custom infrastructure: none. Vibes: immaculate.

How it works

You --> yeet run --> Nomad --> Cheap PC --> AI Agent --> Draft PR --> You (hours later, caffeinated)

That's it. That's the architecture diagram. No, there isn't a second slide.

The yeet CLI dispatches a parameterized Nomad job. Nomad schedules it onto a machine that has the right project cloned and the right USB devices plugged in. A shell script (run-agent.sh) handles git, spawns the AI, commits results, opens a PR, and records cost. Nomad handles health, retries, logs, and fleet management. You handle being elsewhere.

Features

Agent-agnostic

We don't play favorites. Claude, Crush, Aider -- whoever gets the job done. Bring your own agent if you want. We're not your mom.

Device-aware routing

Need a YubiKey plugged in? yeet knows which box has one. It's like Uber but for USB devices.

Cost controls

Set a budget so your AI doesn't spend your rent money on API calls. Capitalism requires guardrails.

Approval gates

If the AI tries something sketchy, yeet pauses and asks you first. Like a responsible adult.

Fleet management

Drain a runner, take it offline, bring it back. Like managing a very small, very cheap data center.

Nomad-powered

We literally just wrapped Nomad's API. Standing on the shoulders of HashiCorp giants. They did the hard part.

Supported runtimes

Runtime Headless Resume Providers
Crush (OpenCode) crush run "..." --session 15+ (Anthropic, OpenAI, Google, etc.)
Claude Code claude -p "..." --resume Anthropic
Aider aider --message "..." Limited Multi-provider

Adding a new runtime means adding a case branch to a shell script. That's the whole plugin system.

CLI

# Yeet a task and forget about it
yeet run peer6 "implement the notification system"

# Pick your weapon
yeet run peer6 "refactor auth" --runtime crush --model anthropic/claude-sonnet-4
yeet run peer6 "refactor auth" --runtime claude --model opus

# Needs hardware? yeet routes to the right Dell.
yeet run login2 "test FIDO2 flow" --needs yubikey

# Check on your fleet of discount workstations
yeet runners

# What's happening right now
yeet status

# Oh no, what's it doing
yeet logs peer6-abc123

# It's 90% done, just fix the edge case
yeet continue peer6-abc123 "also handle the null case in auth"

# STOP STOP STOP
yeet stop peer6-abc123

# How much did that cost me
yeet cost --period week

Every command is a thin wrapper around a Nomad API call. If yeet doesn't do what you need, nomad CLI does. They're interchangeable.

The hardware

You know that old PC in the cupboard? The thin client you decommissioned? The NUC gathering dust? It runs yeet.

Any x86 Linux box
~15W thin clients sip power
~$5/yr electricity per node
USB plug in your devices
24/7 runs forever

The AI runs in the cloud. These boxes just manage repos, shuttle files, and look unassuming. A $50 thin client from eBay has more computing power than the job requires. An old laptop works too. Anything that runs Linux and stays on.
